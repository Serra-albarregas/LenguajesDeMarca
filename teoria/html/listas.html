Listas

Las listas sirven para organizar información.

Listas desdordenadas

Las listas desordenadas tienen por defecto un puntito antes de cada elemento.

Una lista ordenada comienza con la etiqueta <ul>, posteriormente cada elemento comienza con la etiqueta <li>. 


    <ul>
        <li>Gato</li>
        <li>Perrete</li>
        <li>Caballo</li>
        <li>Tortuga</li>
    </ul>
  • Gato
  • Perrete
  • Caballo
  • Tortuga

Listas ordenada

Las listas desordenadas aparecen numeradas. No se debe confundir con un orden alfabético de los elementos, siendo este dependiente del programador.

Una lista ordenada comienza con la etiqueta <ol>, posteriormente cada elemento comienza con la etiqueta <li>. 


    <ol>
        <li>Tarta</li>
        <li>Profiterol</li>
        <li>Helado</li>
        <li>Galleta</li>
    </ol>
  1. Tarta
  2. Profiterol
  3. Helado
  4. Galleta

Listas de definición

Las listas de definición permiten organizar y listar conceptos junto con una descripción.

Una lista ordenada comienza con la etiqueta <dl>, posteriormente cada elemento comienza con la etiqueta <dt>. Finalmente, la definicón del elemento utiliza la etiqueta <dd>.

Coche
Vehículo con 4 ruedas
Moto
Vehículo con 2 ruedas

    <dl>
        <dt>Coche</dt>
        <dd>Vehículo con 4 ruedas</dd>
        <dt>Moto</dt>
        <dd>Vehículo con 2 ruedas</dd>
    </dl>

Listas anidadas

Son listas dentro de listas

Una lista anidada se crea al utilizar una etiqueta de tipo <ul> o <ol> tras cualquier elemento de la lista de nivel superior. 


    <ul>
        <li>Gato</li>
        <ul>
            <li>Siames</li>
            <li>Persa</li>
        </ul>
        <li>Perrete</li>
        <li>Caballo</li>
        <li>Tortuga</li>
    </ul>
  • Gato
    • Siames

  • Perrete
